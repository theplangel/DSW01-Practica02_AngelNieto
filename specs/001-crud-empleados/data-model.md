# Data Model – CRUD de empleados

## Entities

### Empleado
| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| id | UUID | PK, generated by DB | Opcional si se usa surrogate key además de `clave`; facilita integridad referencial.
| clave | VARCHAR(100) | NOT NULL, UNIQUE index on `upper(clave)` | Debe iniciar con `E-` seguido de 1-97 caracteres alfanuméricos, se normaliza a mayúsculas previo a persistencia.
| nombre | VARCHAR(100) | NOT NULL | Nombre completo del empleado.
| direccion | VARCHAR(100) | NOT NULL | Dirección registrada (sin formato libre mayor a 100 chars).
| telefono | VARCHAR(100) | NOT NULL | Número de contacto almacenado como cadena para permitir extensiones.
| created_at | TIMESTAMP WITH TIME ZONE | NOT NULL, default now() | Facilita auditoría básica.
| updated_at | TIMESTAMP WITH TIME ZONE | NOT NULL, default now(), updated on change | Permite verificar consistencia tras actualizaciones.

## Relationships

- Modelo auto contenido: no hay relaciones con otras tablas en este feature.
- Índice adicional recomendado sobre `clave` para acelerar búsquedas exactas.

## Validation Rules

1. `clave`, `nombre`, `direccion`, `telefono` son obligatorios en creación y actualización.
2. Longitud máxima de 100 caracteres para `nombre`, `direccion` y `telefono`; rechazar cadenas vacías.
3. `clave` debe cumplir el patrón `E-[A-Z0-9]{1,97}`; se convierte a mayúsculas antes de verificar unicidad insensible a mayúsculas.
4. Operaciones de actualización sólo permiten modificar `nombre`, `direccion`, `telefono`.
5. Eliminaciones son físicas (`DELETE FROM empleados WHERE clave = :clave`).

## State Transitions

| State | Trigger | Next State | Notes |
|-------|---------|------------|-------|
| inexistente | `POST /api/v1/empleados` con datos válidos y clave libre | activo | Registra nuevo empleado y setea timestamps.
| activo | `PUT /api/v1/empleados/{clave}` con datos válidos | activo | Actualiza campos editables y `updated_at`.
| activo | `DELETE /api/v1/empleados/{clave}` | eliminado | Registro removido definitivamente; consultas posteriores regresan "no encontrado".

## Derived Behaviors

- Listados devuelven colección ordenada por `clave` ascendente para coherencia de UI.
- Los errores de duplicado se traducen a HTTP 409 con mensaje estándar.
- Las búsquedas puntuales fallidas emiten HTTP 404 sin detallar si la `clave` existió previamente.
